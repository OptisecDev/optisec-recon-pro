🔑 المصادقة — Authentication

All API endpoints require a valid JWT token. Obtain one via:

POST /api/auth/login
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"Optisec123!"}'
Response
{
  "access_token": "eyJhbGci...",
  "token_type": "bearer",
  "user": {"id":1,"username":"admin","role":"admin"}
}
Use the token
curl http://localhost:8000/api/scans \
  -H "Authorization: Bearer eyJhbGci..."
⚡ Quick Start — Launch a Scan
1 Login → get token
2 Launch scan
POST /api/scan
{
  "target": "tesla.com",
  "scan_types": ["xss","sqli","subdomain"]
}
3 Poll for results
GET /api/scan/{scan_id}
{"status":"running","progress":58,...}
4 Or use WebSocket for live updates
ws://localhost:8000/ws/{scan_id}
{"type":"progress","step":"xss","progress":58}

🔄 WebSocket — Real-time Scan

Connect to ws://host/ws/{scan_id} to receive live events:

// Progress update
{"type":"progress","step":"xss","progress":58,"status":"running"}

// Completed
{"type":"done","progress":100,"status":"done","results":{...}}

// Error
{"type":"error","error":"Connection refused"}

Events fire after each module completes. The connection closes automatically when the scan finishes.
